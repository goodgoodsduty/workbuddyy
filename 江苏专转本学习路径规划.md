**【重点必考】**
- 两个重要极限：lim(x→0) sinx/x = 1，lim(x→∞) (1+1/x)^x = e

**【学习方法】**
1. 先把基本初等函数图像画一遍，建立直观感觉
2. 极限计算是核心，每天练5-10道极限题
3. 等价无穷小代换是速算利器，背熟8个常用等价无穷小
4. 间断点分类用流程图记忆：可去→跳跃→无穷→振荡

## 八、学习方法与备考策略

### 8.1 高等数学复习方法

**三轮复习法：**

| 轮次 | 时间 | 任务 | 目标 |
|------|------|------|------|
| 第一轮 | 大二上+大二下 | 过教材+做课后题 | 理解概念，会做基础题 |
| 第二轮 | 暑假+大三上 | 专题训练+总结方法 | 掌握各类题型解法 |
| 第三轮 | 大三下 | 真题+模拟 | 限时答题，查漏补缺 |

**做题原则：**
1. **先懂后做**：看懂例题再动手，不要边看答案边写
2. **限时训练**：平时做题就掐时间，计算题每题不超过10分钟
3. **错题必究**：每道错题标注错误原因（概念错/计算错/方法错）
4. **公式默写**：每周默写一次公式表，包括积分表和展开式

### 8.2 专业课复习方法

**理论课（计算机应用基础+信息技术导论）：**
1. **以大纲为纲**：对照大纲逐条过，不超纲不遗漏
2. **思维导图**：每章画一张思维导图，理清知识框架
3. **关键词记忆**：用关键词串联知识点，如OSI七层"物数网传会表应"
4. **刷真题**：真题重复率高，至少刷3遍

**操作技能课：**
1. **C语言**：每天手写代码（考试是笔试！），背熟常见算法模板
2. **MySQL**：在phpStudy环境里反复练SQL语句，手写也要练
3. **Linux**：虚拟机装CentOS 7，常用命令练到条件反射
4. **Windows**：实际操作一遍每个管理功能
5. **UML**：练10-20道画图题，掌握用例图和类图的标准画法
6. **局域网**：用eNSP模拟器练习网络配置

### 8.3 英语复习方法（基础薄弱版）

**词汇三遍法：**
1. 第一遍：每天30个新词，只求混个眼熟
2. 第二遍：每天50个复习+20个新词，能认出意思
3. 第三遍：每天80个滚动复习，能拼写+会用

**语法套路：**
1. 先学五大基本句型，这是所有语法的基础
2. 时态语态用时间轴画图记忆
3. 从句先搞懂"引导词+语序"，再练翻译
4. 非谓语动词是难点，多做对比练习

**阅读方法：**
1. 先读题目，带着问题读文章
2. 标注文章中的关键句（首尾句、转折句）
3. 不会的词不要停，根据上下文猜意思
4. 每篇阅读控制在8-10分钟

### 8.4 笔记体系

**三本笔记法：**

| 笔记本 | 内容 | 更新频率 |
|--------|------|---------|
| 知识笔记 | 各科知识点框架、公式、概念 | 随学随记 |
| 错题本 | 做错的题+错误原因+正确解法 | 每周整理 |
| 考点本 | 真题高频考点+自己的薄弱点 | 每月更新 |

### 8.5 模拟考试节奏

| 阶段 | 频率 | 内容 |
|------|------|------|
| 大二下 | 每月1次 | 单科模拟（高数/英语/专业课） |
| 暑假 | 每两周1次 | 单科模拟 |
| 大三上 | 每周1次 | 单科模拟 + 每月1次全科模拟 |
| 考前1月 | 每周2次 | 全科模拟（严格按考试时间） |

### 8.6 每日时间管理模板

```
06:30-07:00  起床洗漱
07:00-08:00  英语单词 + 语法（1h）
08:00-12:00  上课（专业课认真听，和备考结合）
12:00-14:00  午休 + 碎片时间刷单词
14:00-17:00  上课/自习
17:00-18:00  晚饭 + 休息
18:30-20:30  高数（2h）
20:30-22:00  专业课/C语言（1.5h）
22:00-22:30  当天复习 + 错题整理（0.5h）
22:30         睡觉
```

### 8.7 心态管理

1. **别和别人比进度**：按自己的节奏走，基础不同起点不同
2. **别贪多**：宁可少学一点学扎实，也不要囫囵吞枣
3. **定期复盘**：每周日晚花30分钟回顾本周学了多少，调整下周计划
4. **适当休息**：每周留半天完全休息，保持长期战斗力
5. **相信自己**：从大二开始准备，时间完全够用，关键是坚持

## 九、关键提醒

### 9.1 容易踩的坑

| 坑 | 解决方案 |
|----|---------|
| 高数只看不练 | 每天必须动手做题，看懂≠会做 |
| 专业课死记硬背 | 先理解再记忆，用思维导图串框架 |
| C语言只上机不手写 | 考试是笔试！每周至少手写3个完整程序 |
| 英语单词背了就忘 | 用艾宾浩斯曲线复习，第二天必复习前一天 |
| 真题留到最后做 | 真题至少刷3遍，早点接触了解命题规律 |
| 忽略大纲 | 大纲是唯一标准，超纲内容不用管 |

### 9.2 你的优势

作为IT/网络管理专业，你有天然优势：
- **C语言**：可能已经学过，复习比零基础快
- **MySQL**：做过PHP+MySQL登录系统，有实操基础
- **局域网**：有IIS服务器配置经验，网络知识不陌生
- **Linux**：如果接触过命令行，起步更快
- **Windows**：日常使用，配置管理不陌生

**把这些优势发挥好，专业课操作技能部分可以少花时间，多分给高数和英语。**

### 9.3 优先级排序（基础薄弱版）

| 优先级 | 科目 | 理由 |
|--------|------|------|
| 1 | 高数(150分) | 分值高，提分空间大，方法固定 |
| 2 | 专业理论(150分) | 分值高，有专业基础，记忆为主 |
| 3 | 英语(120分) | 基础薄弱需长期积累，不能突击 |
| 4 | 操作技能(80分) | 有专业基础，短期可突破 |
