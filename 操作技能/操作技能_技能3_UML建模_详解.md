## 一、UML 概述

### 1.1 UML 是什么

**【UML】** Unified Modeling Language（统一建模语言）

**【作用】** 软件系统可视化建模的标准语言

| | 组件图 | 组件关系 |
| | 部署图 | 物理部署 |
| | 包图 | 包结构 |
| **行为图** | 用例图 | 功能需求 |
| | 活动图 | 工作流 |
| | 状态图 | 状态变化 |

## 二、用例图（Use Case Diagram）⚠️ 必考

### 2.1 组成元素

| 元素 | 图形 | 说明 |
|------|------|------|
| **参与者（Actor）** | 小人图标 | 与系统交互的人或外部系统 |
| **用例（Use Case）** | 椭圆 | 系统提供的功能 |
| **系统边界** | 矩形 | 系统的范围 |
| **关系** | 线条 | 元素间关系 |

### 2.2 用例图示例

```
         客户                管理员
          |                    |
     (查询商品)           (管理商品)
     (购买商品)           (管理订单)
     (查看订单)
          |                    |
     ┌────┴────────────────────┴────┐
     │       网上购物系统           │
     └──────────────────────────────┘
```

### 2.3 用例之间的关系 ⚠️ 必考

| 关系 | 图形 | 说明 |
|------|------|------|
| **关联（Association）** | 实线 | 参与者与用例之间 |
| **包含（Include）** | 虚线+箭头+<<include>> | A包含B（A→B）|
| **扩展（Extend）** | 虚线+箭头+<<extend>> | B扩展A（B→A）|
| **泛化（Generalization）** | 实线+空心三角 | 继承关系 |

**【Include vs Extend 区别】** ⚠️

| 关系 | 含义 | 例 |
|------|------|---|
| **Include** | A 必须执行 B | 登录必须包含验证身份 |
| **Extend** | A 可能执行 B | 购买可能扩展为打折 |

### 2.4 画用例图步骤

1. 确定系统边界
2. 找出参与者（人、外部系统）
3. 找出用例（系统功能）
4. 确定关系（关联/包含/扩展）
5. 画图

### 2.5 例题

**【题目】** 画一个ATM机系统的用例图，包括：客户（取款、查询余额、修改密码）、银行（维护系统）

**【解答】**

```
            客户
             |
        ┌────┴────┐
       (取款)  (查询余额)
        (修改密码)
        ┌─────────┘
        ↓
   <<include>>
        ↓
     (验证密码)
```

## 三、类图（Class Diagram）⚠️ 必考

### 3.1 类的表示

```
┌──────────────────┐
│     类名         │
├──────────────────┤
│ - 属性1: 类型     │
│ - 属性2: 类型     │
├──────────────────┤
│ + 方法1(): 返回  │
│ + 方法2(): 返回  │
└──────────────────┘
```

### 3.2 访问修饰符

| 符号 | 含义 |
|------|------|
| + | public（公共）|
| - | private（私有）|
| # | protected（保护）|
| ~ | package（包）|

### 3.3 类之间的关系 ⚠️ 必考

| 关系 | 图形 | 含义 | 例子 |
|------|------|------|------|
| **关联（Association）** | 实线 | 类之间有联系 | 老师-学生 |
| **聚合（Aggregation）** | 空心菱形 | 整体-部分（弱）| 班级-学生 |
| **组合（Composition）** | 实心菱形 | 整体-部分（强）| 人-心脏 |
| **泛化（Generalization）** | 空心三角 | 继承 | 动物-狗 |
| **实现（Realization）** | 虚线+空心三角 | 实现接口 | 类-接口 |
| **依赖（Dependency）** | 虚线+箭头 | 一个类用另一个 | A-临时用B |

### 3.4 关系详解

**【关联】** 普通联系
- 单向关联：A → B
- 双向关联：A ↔ B
- 自关联：A → A

**【聚合】** 整体与部分，但部分可以独立存在
- 班级（整体）有学生（部分）
- 学生离开班级还能存在

**【组合】** 整体与部分，部分不能独立
- 人（整体）有心脏（部分）
- 心脏离开人无法存在

**【泛化（继承）】** 子类继承父类

**【实现】** 类实现接口

**【依赖】** 一个类用另一个类作为参数

### 3.5 类图示例

```
        ┌──────────────┐
        │    动物      │  ← 父类
        ├──────────────┤
        │ - 名字        │
        │ - 年龄        │
        ├──────────────┤
        │ + 吃东西()    │
        └──────────────┘
              △
              │ 泛化
        ┌─────┴─────┐
        │            │
   ┌────┴────┐  ┌────┴────┐
   │   狗    │  │   猫    │
   ├─────────┤  ├─────────┤
   │ - 品种  │  │ - 颜色  │
   ├─────────┤  ├─────────┤
   │ + 叫()  │  │ + 叫()  │
   └─────────┘  └─────────┘
```

### 3.6 例题

**【题目】** 设计一个学生选课系统的类图：
- 学生：学号、姓名、选课
- 课程：课号、课名、学分
- 关系：学生选课是多对多

**【解答】**

```
┌──────────────┐         ┌──────────────┐
│   Student    │         │   Course     │
├──────────────┤         ├──────────────┤
│ - sid: int   │         │ - cid: int   │
│ - name: str  │   n   m │ - cname: str │
├──────────────┤ ────── ├──────────────┤
│ + 选课()     │         │ + 添加学生() │
└──────────────┘         └──────────────┘
```

## 四、其他UML图（了解）

### 4.1 顺序图

**【元素】**
- 对象（矩形）
- 生命线（虚线）
- 激活（条形）
- 消息（箭头）

```
  客户          系统           银行
   │            │              │
   │──请求登录──→│              │
   │            │──验证身份──→│
   │            │←─通过────────│
   │←─登录成功──│              │
   │            │              │
```

### 4.2 状态图

**【元素】**
- 状态（圆角矩形）
- 转换（箭头）
- 初始/结束（实心圆/双圆）

```
  ●→ [空闲] ──开始订单──→ [处理中] ──完成──→ [完成] → ◉
                ↓ 失败
            [失败] → ◉
```

### 4.3 活动图

**【元素】**
- 起点/终点
- 活动（圆角矩形）
- 判断（菱形）
- 分叉/汇合
