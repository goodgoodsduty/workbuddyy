## 一、OSPF基础与虚连接配置

### 1. 拓扑结构（典型考法）

```
                    Area 0（骨干区域）
                         │
    R1 ──────── R2(ABR) ──────── R3(ABR)
    │           │                 │
  Area 1      Area 2            Area 3
   │                             │
  PC1                            PC2
```

### 2. 配置框架

```bash
# ====== 所有路由器通用步骤 =======

# ① 开启OSPF进程
ospf 1 router-id x.x.x.x

# ② 宣告网段进入对应区域
area 0
  network x.x.x.x y.y.y.y      # 接口IP + 反掩码
area 1
  network x.x.x.x y.y.y.y

# ③ （可选）修改接口开销/优先级
interface g0/0/0
  ospf cost 10                  # 修改开销值
  ospf dr-priority 100          # 修改DR选举优先级
```

### 3. 虚连接配置（重点！）

**使用场景**：非骨干区域没有直连Area 0时，用虚连接打通

```bash
# 在两个ABR上分别配置（双向都要配）

# R2上的配置
ospf 1
  area 2                        # 中间的非骨干过渡区域
    vlink-peer 3.3.3.3          # 对端ABR的Router-ID

# R3上的配置（对称配置）
ospf 1
  area 2
    vlink-peer 2.2.2.2          # 对端ABR的Router-ID
```

### 4. 验证命令

```bash
display ospf peer              # 查看邻居关系（Full状态才算通）
display ospf interface          # 查看接口OSPF信息
display ip routing-table protocol ospf   # 查看OSPF路由表
display ospf vlink             # 查看虚连接状态
display ospf lsdb             # 查看链路状态数据库
ping 对端IP                    # 测试连通性
```

### 5. 常见错误排查

| 问题 | 原因 | 解决 |
|------|------|------|
| 邻居卡在Init | Hello间隔/ Dead时间不一致 | 统一两端参数 |
| 邻居卡在2-way | 以太网DR/BDR选举问题 | 正常现象或检查MTU |
| 邻居一直Down | 区域ID不匹配 | 检查area号 |
| 虚连接不通 | 过渡区域必须是普通区域 | 不能是Stub/NSSA |
| 学不到路由 | 网络宣告漏了或反掩码错 | 检查network语句 |

## 二、STP配置（生成树协议）

### 1. 拓扑结构

```
        SW1（核心交换机）
       /     \
     /         \
   SW2          SW3
    |            |
   PC1          PC2
```

### 2. 配置框架

```bash
# ====== 方法一：STP基本配置 =======

# ① 全局开启STP
stp enable

# ② 设置STP模式（华为默认MSTP）
stp mode stp                     # 或 mstp / rstp
# 推荐：stp mode rstp            # RSTP收敛更快

# ③ 指定根桥（选一台做根桥）
stp root primary                 # 主根桥
# 另一台：
stp root secondary               # 备份根桥

# ④ 修改端口优先级（影响根端口选择）
stp port priority 128            # 默认128，数值越小越优先
```

```bash
# ====== 方法二：MSTP多实例配置（进阶）======

# ① 进入MSTP域视图
stp region-configuration
  region-name huawei            # 域名（必须所有交换机一致）
  revision-level 1              # 修订级别（一致）
  instance 1 vlan 10 20         # 实例1映射VLAN 10,20
  instance 2 vlan 30 40         # 实例2映射VLAN 30,40
  active region-configuration   # 激活配置
quit

# ② 设置各实例的根桥
stp instance 1 root primary     # 本机为实例1的主根桥
stp instance 2 root secondary   # 本机为实例2的备份根桥

# ③ 开启STP
stp enable
```

### 3. 边缘端口配置（重要！接PC的口要配）

```bash
interface e0/0/1                # 接PC的接口
  stp edged-port enable         # 设为边缘端口，跳过计算，立即转发
  stp bpdu-filter enable        # 可选：不收发BPDU（更安全）
```

### 4. 验证命令

```bash
display stp                    # 查看STP全局摘要信息
display stp brief              # 各端口STP角色/状态简览
display stp instance 1         # 查看指定实例的STP信息
display stp root              # 查看根桥信息
```

### 5. STP端口角色与状态

| 角色 | 说明 |
|------|------|
| RP（Root Port）| 根端口，离根桥最近的口 |
| DP（Designated Port）| 指定端口，负责转发 |
| AP（Alternate Port）| 替代端口，阻塞备份 |
| BP（Blocked Port）| 阻塞端口，不转发 |

| 状态 | 说明 |
|------|------|
| Forwarding | 转发数据 |
| Learning | 学习MAC地址 |
| Blocking/Discarding | 不转发 |

## 三、基于接口划分VLAN

### 1. 拓扑结构

```
    PC1(VLAN10) ──┐
                   ├── SW1 ──── SW2 ──── PC3(VLAN20)
    PC2(VLAN10) ──┘              │
                               PC4(VLAN30)
```

### 2. 配置框架（Access口 - 接PC）

```bash
# ====== 创建VLAN ======
vlan batch 10 20 30             # 批量创建多个VLAN
# 或单独创建
vlan 10
  name sales                    # 可选：命名
quit

# ====== Access接口（接PC）======
interface g0/0/1
  port link-type access         # 接入模式
  port default vlan 10          # 划入VLAN 10

interface g0/0/2
  port link-type access
  port default vlan 20
```

### 3. Trunk口配置（交换机之间级联）

```bash
interface g0/0/24               # 连接另一台交换机的口
  port link-type trunk          # Trunk模式
  port trunk allow-pass vlan all # 允许所有VLAN通过
  # 或指定允许的VLAN
  # port trunk allow-pass vlan 10 20 30
```

### 4. 批量划分VLAN（端口组）

```bash
# ① 批量将g0/0/1到g0/0/5划入VLAN 10
port-group group-member g0/0/1 to g0/0/5
  port link-type access
  port default vlan 10

# ② 批量将g0/0/6到g0/0/10划入VLAN 20
port-group group-member g0/0/6 to g0/0/10
  port link-type access
  port default vlan 20
```

### 5. Hybrid口配置（特殊场景）

```bash
interface g0/0/10
  port link-type hybrid         # 混合模式（华为特有）
  port hybrid pvid vlan 10      # PVID（本机VLAN标签）
  port hybrid untagged vlan 10  # 去标签发出（给PC）
  port hybrid tagged vlan 20 30 # 带标签发出（给下游交换机）
```

### 5. 验证命令

```bash
display vlan                   # 查看所有VLAN及成员接口
display port vlan             # 查看接口VLAN归属
display vlan 10               # 查看指定VLAN详细信息
display interface brief        # 查看接口简要信息
ping 同VLAN内其他PC IP         # 测试同VLAN互通
```

### 6. VLAN类型对比

| 类型 | 用途 | 发出帧是否带Tag |
|------|------|---------------|
| Access | 接PC/终端 | 不带Tag |
| Trunk | 交换机间级联 | 除PVID外带Tag |
| Hybrid | 灵活控制 | 可按VLAN指定 |

## 四、静态路由配置综合案例

### 1. 典型拓扑（三台路由器互连）

```
    192.168.12.0/24
  R1 ───────────── R2 ───────────── R3
  │192.168.1.0/24  │192.168.23.0/24  │192.168.3.0/24
  PC1               PC2              PC3
```

### 2. 配置框架

```bash
# ====== 第一步：配置接口IP地址 ======

# R1配置
interface g0/0/0
  ip address 192.168.12.1 24        # 连R2的接口
interface g0/0/1
  ip address 192.168.1.254 24      # 连PC1的接口（网关）

# R2配置
interface g0/0/0
  ip address 192.168.12.2 24        # 连R1的接口
interface g0/0/1
  ip address 192.168.23.2 24        # 连R3的接口

# R3配置
interface g0/0/0
  ip address 192.168.23.3 24        # 连R2的接口
interface g0/0/1
  ip address 192.168.3.254 24      # 连PC3的接口（网关）
```

```bash
# ====== 第二步：配置静态路由 ======

# R1：要去往192.168.3.0/24和192.168.23.0/24，下一跳是R2
ip route-static 192.168.3.0 24 192.168.12.2
ip route-static 192.168.23.0 24 192.168.12.2

# R2：（直连网段不需要配，但如果是中间设备可能需要）
ip route-static 192.168.1.0 24 192.168.12.1    # 去往R1后缀
ip route-static 192.168.3.0 24 192.168.23.3    # 去往R3后缀

# R3：要去往192.168.1.0/24和192.168.12.0/24，下一跳是R2
ip route-static 192.168.1.0 24 192.168.23.2
ip route-static 192.168.12.0 24 192.168.23.2
```

```bash
# ====== 第三步：配置默认路由（可选，简化配置）======

# 如果只有一个出口，可以只配一条默认路由
ip route-static 0.0.0.0 0 192.168.12.2     # R1默认走R2
ip route-static 0.0.0.0 0 192.168.23.2     # R3默认走R2
```

```bash
# ====== 第四步：浮动静态路由（备份链路）======
# 主路由断开时自动启用备份路由

# R1主路由（优先级60，默认）
ip route-static 192.168.3.0 24 192.168.12.2 preference 60

# R1备份路由（优先级100，主断了才生效）
ip route-static 192.168.3.0 24 192.168.13.2 preference 100
```

### 3. 验证命令

```bash
display ip routing-table          # 查看完整路由表
display ip routing-table protocol static  # 只看静态路由
ping 192.168.3.1                  # 从PC1 ping PC3（跨网段测试）
tracert 192.168.3.1               # 跟踪路由路径
display interface brief           # 检查接口UP/DOWN状态
```

### 4. 静态路由 vs 动态路由

| 对比项 | 静态路由 | OSPF动态路由 |
|--------|---------|-------------|
| 配置复杂度 | 低 | 高 |
| 适用规模 | 小型网络（<10台路由器） | 中大型网络 |
| 维护成本 | 手动维护，拓扑变了要改 | 自动学习，自适应变化 |
| 带宽占用 | 无 | 占少量带宽发报文 |

## 五、通用配置流程（万能模板）

每道题拿到后按这个顺序来，不会漏：

```
第1步：画拓扑图，标注IP/VLAN/区域
  ↓
第2步：配置各设备接口IP地址（最基础！）
  ↓
第3步：根据题目要求选择技术（OSPF/STP/VLAN/静态路由）
  ↓
第4步：按框架逐条输入命令（先全局→再接口）
  ↓
第5步：保存配置（save / write）
  ↓
第6步：验证（ping/tracert/display命令）
  ↓
第7步：截图留证（考试要求）
```

## 六、eNSP模拟器常用快捷键

| 操作 | 快捷键/说明 |
|------|------------|
| 启动所有设备 | 工具栏 → "启动"按钮 |
| 打开CLI终端 | 双击设备图标 |
| 退出用户视图 | `quit` 返回上级 |
| 进入系统视图 | `system-view` 简写 `sys` |
| 保存配置 | 在用户视图下输入 `save` |
| 查看当前配置 | `display current-configuration` |
| 删除某行配置 | 前面加 `undo`，如 `undo ip route-static ...` |
| 清空配置重装 | `reset saved-configuration` → 重启 |
